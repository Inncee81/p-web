     <h1>
     </h1>
     <div id="content">
      <p style="margin-top: 2em; margin-bottom: 2em;">
       Security, speed, compliance, and flexibility -- all of these describe lighttpd (
       <i>
        pron.
       </i>
       <b>
        lighty
       </b>
       ) which is rapidly redefining efficiency of a webserver; as it is designed and optimized for high performance environments. With a small memory footprint compared to other web-servers, effective management of the cpu-load, and advanced feature set (FastCGI, SCGI, Auth, Output-Compression, URL-Rewriting and many more) lighttpd is the perfect solution for every server that is suffering load problems. And best of all it's Open Source licensed under the
       <a href="http://www.lighttpd.net/assets/COPYING">
        revised BSD license
       </a>
       .
      </p>
      <h2>
       Web 2.0
      </h2>
      <p>
       lighttpd
       <a href="http://redmine.lighttpd.net/projects/lighttpd/wiki/PoweredByLighttpd">
        powers
       </a>
       several popular Web 2.0 sites. Its
	high speed io-infrastructure allows them to
       <a href="http://www.lighttpd.net/benchmark">
        scale several times better
       </a>
       with the same hardware than with alternative web-servers.
      </p>
      <p>
       This fast web server and its
       <a href="http://redmine.lighttpd.net/projects/lighttpd/wiki/DevelopersList">
        development team
       </a>
       create a web-server with the needs of the future web in mind:
      </p>
      <ul>
       <li>
        <a href="http://blog.lighttpd.net/articles/2006/11/29/faster-fastcgi">
         Faster FastCGI
        </a>
       </li>
       <li>
        <a href="http://blog.lighttpd.net/articles/2006/11/27/comet-meets-mod_mailbox">
         COMET meets mod_mailbox
        </a>
       </li>
       <li>
        <a href="http://blog.lighttpd.net/articles/2006/11/12/lighty-1-5-0-and-linux-aio">
         Async IO
        </a>
       </li>
      </ul>
      Its event-driven architecture is optimized for a large number of parallel connections (keep-alive) which is important for high performance AJAX applications.

        <h2>
         Future Scheduled Behavior Changes
        </h2>
        <ul>
         <li>
          graceful restart/shutdown default timeout will change from
          <br/>
          0 (infinite/no timeout) to 5 seconds (or some similar non-zero period)
          <br/>
          configure an alternative with:
          <br/>
          server.feature-flags += (“server.graceful-shutdown-timeout” =&gt; 5)
         </li>
         <li>
          mod_compress is
          <span class="caps">
           DEPRECATED

          mod_compress has been subsumed by mod_deflate
          <br/>
          Note: mod_compress config options may be removed in a future release
