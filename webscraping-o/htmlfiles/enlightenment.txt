       <h3 id="Window_Manager">
        Window Manager
       </h3>
       <div class="level3">
        <p>
         <img alt="" class="mediaright" src="/_media/shot-enlightenment.png?w=320&amp;tok=42f6c0" width="320"/>
        </p>
        <p>
         <a class="wikilink2" href="/about-enlightenment" rel="nofollow" title="about-enlightenment">
          Enlightenment
         </a>
         started out way back in 1996 as a project to build a Window Manager for X11. It has grown much more since then. Enlightenment still produces this Window Manager, but it has evolved to also cover Mobile, Wearable and TV UI requirements for projects such as
         <a class="urlextern" href="http://www.tizen.org" rel="nofollow" title="http://www.tizen.org">
          Tizen
         </a>
         as well as the traditional “desktop” UI. We still push out releases. See our
         <a class="wikilink1" href="/download" title="download">
          download
         </a>
         page for more details on this. Visit our
         <a class="wikilink1" href="/contrib/start" title="contrib:start">
          contribute
         </a>
         page for our latest source code repositories.
        </p>
        <p>
         The project is currently transitioning from
         <a class="urlextern" href="http://www.x.org" rel="nofollow" title="http://www.x.org">
          X11
         </a>
         to
         <a class="urlextern" href="http://wayland.freedesktop.org" rel="nofollow" title="http://wayland.freedesktop.org">
          Wayland
         </a>
         . We are fully committed to moving to Wayland eventually, as its the future of graphical display layers on Linux.
        </p>
        <p>
         We still primarily support Linux for Enlightenment, but there is some effort (based on help and support from users and some developers) to support BSD too.
        </p>
        <hr/>
       </div>
       <h3 id="Libraries">
        Libraries
       </h3>
       <div class="level3">
        <p>
         <img alt="" class="mediaright" src="/_media/diagram-block-efl.svg?w=320&amp;tok=a503b2" width="320"/>
        </p>
        <p>
         While developing the Window Manager, a set of libraries was also developed. These libraries are known collectively as
         <a class="wikilink2" href="/about-efl" rel="nofollow" title="about-efl">
          EFL
         </a>
         . They cover a range of functionality from main-loop, to graphics, scene graphs, networking, widgets, data storage, IPC and much more.
        </p>
        <p>
         We now are starting to pull in bindings support directly into EFL. We are working on auto-generated bindings for C++, Lua and Javascript (v8/node.js). We hope eventually to pull in our Python bindings too, so we can support as many languages as possible and keep them up to date.
        </p>
        <p>
         Our primary development environment is Linux, but we make an effort to support BSD as much as possible, as well as macOS and Windows.
        </p>
        <hr/>
       </div>
       <h3 id="Applications">
        Applications
       </h3>
       <div class="level3">
        <p>
         <img alt="" class="medialeft" src="/_media/shot-terminology.png?w=240&amp;tok=a8193a" width="240"/>
        </p>
        <p>
         We use our libraries not only to make
         <a class="wikilink2" href="/about-enlightenment" rel="nofollow" title="about-enlightenment">
          Enlightenment
         </a>
         but other applications for day to day use. We make these applications available for free.
        </p>
        <p>
         <img alt="" class="mediaright" src="/_media/shot-rage.png?w=240&amp;tok=f38751" width="240"/>
        </p>
        <p>
         We have included common features such as a
         <a class="wikilink2" href="/about-terminology" rel="nofollow" title="about-terminology">
          terminal emulator
         </a>
         ,
         <a class="wikilink2" href="/about-rage" rel="nofollow" title="about-rage">
          video player
         </a>
         , an
         <a class="wikilink2" href="/about-ephoto" rel="nofollow" title="about-ephoto">
          image viewer
         </a>
         , and even the makings of
         <a class="wikilink2" href="/about-edi" rel="nofollow" title="about-edi">
          an IDE
         </a>
         .
        </p>
        <p>
         <a class="urlextern" href="http://www.tizen.org" rel="nofollow" title="http://www.tizen.org">
          Native Tizen applications
         </a>
         are developed using EFL. This is because we've focused on remaining lean, yet feature-rich. Unlike many traditional toolkits, EFL is based around a scene graph from the ground up. This allows us to seamlessly switch from software rendering to OpenGL or any other mechanism that can be put in a render engine for Evas the canvas scene graph engine. This also permits layer widgets and objects with alpha channels from the ground up without any special coding tricks.
        </p>
        <p>
         <img alt="" class="medialeft" src="/_media/shot-edi.png?w=240&amp;tok=70cc65" width="240"/>
        </p>
        <p>
         This is by no means a complete list of applications and more are on the way. We may not have started the traditional way, but are building our library over time. Tizen also has a library of its own.
        </p>
        <p>
         <img alt="" class="mediaright" src="/_media/shot-2017-08-22_07-54-31.jpg?w=240&amp;tok=252d77" width="240"/>
        </p>
        <hr/>
