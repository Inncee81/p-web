   <div id="dialogIS" title="I-Series NVR Firmware Upgrade Instructions">
    <div class="field field-name-body field-type-text-with-summary field-label-hidden">
     <div class="field-items">
      <div class="field-item even" property="content:encoded">
       <p>
        View the most updated version of this document here:
       </p>
       <p>
        <a href="https://techsupportca.freshdesk.com/en/support/solutions/articles/17000113531-i-series-nvr-firmware-upgrade-instructions">
         https://techsupportca.freshdesk.com/en/support/solutions/articles/17000113531-i-series-nvr-firmware-upgrade-instructions
        </a>
       </p>
       <p>
       </p>
       <p>
        The I-series NVR (such as the DS-7716NI-I4) is one of Hikvision's most popular and feature-rich recorders. As such, many firmware revisions have been introduced over the years to continually ensure the product is compatible with the newest technology available. Due to the many revisions, we recommend that the user closely follows the instructions below in order to reduce the amount of time spent as well as the chance of failure.
       </p>
       <p>
       </p>
       <h3>
        <strong>
         Database Optimization and Repair
        </strong>
       </h3>
       <p>
        As more affordable IP cameras are introduced over time with greater video resolution and data sizes, more efficient database management also becomes necessary. The introduction of firmware v4.0 brought about a new database architecture in order to be futureproof.
       </p>
       <p>
       </p>
       <p>
        After upgrading to v4.X, the recorder database will need to be converted and optimized. If you are experiencing issues where playback is expected but not found, make sure "Database Repair" is performed as indicated in the procedures and scenarios below.
       </p>
       <p>
       </p>
       <h3>
        <strong>
         Preparing the Upgrade
        </strong>
       </h3>
       <p>
        Before proceeding with upgrade, it is recommended that NVR configuration file is exported from the NVR over the network or on to a local USB drive.
       </p>
       <p>
       </p>
       <h3>
        <strong>
         Upgrading from v3.4.92 build 170518 or Older
        </strong>
       </h3>
       <ol>
        <li>
         <span style="line-height:normal">
          <span style="tab-stops:list .5in">
           All recorders must reach v3.4.92 before proceeding further. Upgrading from versions before v3.4.92 directly to any version of v4.X will likely cause the recorder to fail.
          </span>
         </span>
        </li>
        <li>
         <span style="line-height:normal">
          <span style="tab-stops:list .5in">
           If the recorder is already at v3.4.92, a full factory default is highly recommended before upgrading to any version of v4.X. There is a high chance of unit failure (requiring RMA) if the unit is not defaulted before upgrade.
          </span>
         </span>
        </li>
        <li>
         <span style="line-height:normal">
          <span style="tab-stops:list .5in">
           After reaching v3.4.92 and performing a full factory default, an upgrade directly to v4.50.00 is acceptable.
          </span>
         </span>
        </li>
        <li>
         <span style="line-height:normal">
          <span style="tab-stops:list .5in">
           After the upgrade is completed and the recorder is reprogrammed, it may be beneficial to perform a Database Repair. For details, refer to the section "Database Optimization and Repair" above.
          </span>
         </span>
        </li>
        <li>
         <span style="line-height:normal">
          <span style="tab-stops:list .5in">
           To verify repair progress, you may refer to the HDD status, or search the recorder log for repair started and stopped entries. Note that while the HDD is repairing, new recordings are still being made, but some existing recordings may not be searchable until repair is complete.
          </span>
         </span>
        </li>
        <li>
         <span style="line-height:normal">
          <span style="tab-stops:list .5in">
           If you continue to observe playback issues after database repair, ensure there are no power, network, or motion detection issues. Should the problem persist, contact technical support.
          </span>
         </span>
        </li>
       </ol>
       <p>
       </p>
       <h3>
        <strong>
         Upgrading from Any v4.X Build to v4.50.00.
        </strong>
       </h3>
       <ol>
        <li>
         <span style="line-height:normal">
          <span style="tab-stops:list .5in">
           Any v4.X build can be upgraded directly to v4.50.00.
          </span>
         </span>
        </li>
        <li>
         <span style="line-height:normal">
          <span style="tab-stops:list .5in">
           Export configuration is highly recommended before performing the upgrade.
          </span>
         </span>
        </li>
        <li>
         <span style="line-height:normal">
          <span style="tab-stops:list .5in">
           If upgrading from any v4.X version that was not v4.22.005, a Database Repair is recommended. Refer to Step 4 and onwards in the previous section.
          </span>
         </span>
        </li>
       </ol>
       <p>
       </p>
       <h3>
        <strong>
         Downgrading
        </strong>
       </h3>
       <p>
        Downgrading is not recommended. Due to new features and parameters constantly being added, downgrading may cause the NVR to factory default itself or require a manual default to operate properly.
       </p>
